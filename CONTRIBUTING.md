## Visual Genome Additions

### `vg/vg_preprocess.py`
Converts raw VG JSON into a generalisation split for the GNN.
- Note: vg_processed pkl on drive
- Link to Visual Genome data download: https://homes.cs.washington.edu/~ranjay/visualgenome/api.html
- Dataset Description: https://homes.cs.washington.edu/~ranjay/visualgenome/index.html

**Pipeline:**
1. Streams `vg/data/relationships.json` and `vg/data/objects.json` to JSONL (one image per line)
2. Maps VG WordNet synsets to Hoiverse category IDs via `mappings.py`
3. Builds per-image graph dicts (node features, edge geometry, edge index)
4. Computes normalisation statistics (mean / std) across all graphs
5. Saves a pickle to `vg/processed_data/vg_processed`

**Key design decisions:**
- Uses per-image dimensions from `vg_image_data.py` (not hardcoded 640×480)
- Local dimension-aware feature functions (`vg_bbox_to_features`, `vg_node_features`, `vg_edge_geometry`)
- `MAX_IMAGES = None` processes all images; set to an integer for quick testing

### `vg/evaluate_vg.py`
Runs the trained GNN on the VG generalisation split and saves side-by-side visualisations.

**Key features:**
- **Before panel** — graph topology only, all nodes shown
- **After panel** — GNN-predicted relation labels; only nodes that participate in at least one edge are shown
- `filter_edges(edges, edge_labels, max_edges_per_node=3)` caps edges per node to reduce clutter
- Label collision avoidance: perpendicular nudge up to 8 steps of 18 px
- S/O role badges (coloured circle) placed near arrow tail and head
- Background images fetched at native resolution from VG URLs via `vg_image_data.get_image_url`

- Note: All Visualizations are in the Drive with ones picked out for presentation purposes. Feel free to filter through or generate more visualizations (up to 20,000 visualizations possible).

### `vg/modules/vg_image_data.py`
Single authoritative lookup for VG image metadata loaded from `vg/data/image_data.json`: image urls and image dimensions.

| Function | Returns |
|---|---|
| `get_image_info(image_id)` | Full metadata dict or `None` |
| `get_image_size(image_id)` | `(width, height)`, fallback `(640, 480)` |
| `get_image_url(image_id)` | URL string or `None` |

### `vg/mappings.py`
`VG_TO_HOI` dict mapping VG WordNet synsets (e.g. `"dog.n.01"`) to Hoiverse category names. Extend this to improve VG coverage (Manual process required).

## Running the Pipeline

```bash
# 1. Preprocess VG data into graphs
python vg/vg_preprocess.py

# 2. Run GNN inference and generate visualisations
python vg/evaluate_vg.py
```

Output PNGs are saved to `vg/visualizations/<image_id>.png`.

## Path Conventions

Both `vg_preprocess.py` and `evaluate_vg.py` derive all paths from `__file__` so they work correctly regardless of the working directory:

```python
_VG_DIR   = os.path.dirname(os.path.abspath(__file__))   # .../vg/
_ROOT_DIR = os.path.dirname(_VG_DIR)                      # project root
```

Root-level modules (`GNN`, `preprocess`) are imported after inserting `_ROOT_DIR` into `sys.path`.

## Adding New Relation Classes

1. Add the relation name to `USEFUL_RELATIONS` in `preprocess.py`
2. Add a colour entry in `RELATION_COLORS` in `vg/evaluate_vg.py`
3. Extend `VG_TO_HOI` in `vg/mappings.py` if the new class requires new synset mappings
4. Re-run `vg_preprocess.py`
